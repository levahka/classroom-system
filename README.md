**Программный комплекс для управления и мониторинга компьютерных классов учебных заведений**

ClassroomSystem — это клиент-серверная система, разработанная для контроля компьютерных классов в образовательных учреждениях. Комплекс обеспечивает авторизацию студентов на рабочих станциях, отслеживание сессий, мониторинг запущенных приложений (в том числе игр и социальных сетей), а также удалённое управление процессами на клиентских ПК.

## Архитектура

Система построена по клиент-серверной модели и состоит из трёх Windows-приложений на стороне клиента (рабочие станции учащихся) и серверной СУБД MySQL.

## Компоненты

### ClassroomLogin

**Тип:** WPF-приложение (.NET Framework 4.7.2)

Окно авторизации, которое отображается при входе пользователя в систему. Студент выбирает учебную группу, фамилию и при необходимости сообщает о неисправностях рабочего места.

**Основные функции:**

- Выбор группы и фамилии из выпадающих списков (данные загружаются из БД)
- Оффлайн-режим: при отсутствии связи с БД данные берутся из локального кеша (`students_cache.json`), а поля ввода переключаются в текстовый режим
- Формирование сессии с уникальным идентификатором (формат: `ИМЯ-ПК_ДатаВремя`)
- Запись сессии в локальный журнал (`journal.csv`) и в базу данных
- Передача информации о проблемах рабочего места (быстрые кнопки: мышь, клавиатура, монитор, интернет, звук, «тормозит»; свободный текст до 500 символов)
- Защита от закрытия окна до успешной авторизации: перехват клавиатурных сочетаний (Alt+Tab, Alt+F4, Win, Ctrl+Esc), блокировка панели задач, принудительный вывод окна на передний план

**Механизмы защиты:**

Приложение устанавливает низкоуровневый клавиатурный хук (`SetWindowsHookEx` / `LowLevelKeyboardProc`), блокирующий системные сочетания клавиш до завершения авторизации. Панель задач скрывается, окно фиксируется поверх всех окон. Таймер watchdog каждые 2 секунды проверяет, что окно не было обойдено.

### SessionWatcher

**Тип:** Windows Forms-приложение, работающее в скрытом режиме (.NET Framework 4.7.2)

Фоновый процесс, запускаемый после авторизации. Отвечает за мониторинг пользовательской сессии и активности.

**Основные функции:**

- **Heartbeat-механизм** — каждые 30 секунд отправляет в БД подтверждение активности сессии (IP-адрес, длительность)
- **Мониторинг приложений** — каждые 10 секунд сканирует запущенные процессы и заголовки окон, сопоставляя их со списком отслеживаемых приложений из таблицы `tracked_apps`
- **Удалённые команды** — каждые 10 секунд опрашивает таблицу `remote_commands` и выполняет команды: завершение процесса по имени (`KILL_PROCESS`), завершение по заголовку окна (`KILL_BY_TITLE`), показ сообщения (`MESSAGE`), блокировка экрана (`LOCK_SCREEN`)
- **Offline-first архитектура** — при потере связи с БД все операции сохраняются в локальную очередь (`sync_queue.json`) и отправляются при восстановлении подключения (повторная попытка каждые 120 секунд)
- **Перехват системных событий** — через скрытую форму (`HiddenForm`) отслеживает WM_QUERYENDSESSION, WM_ENDSESSION, WM_POWERBROADCAST для корректного закрытия сессии при выключении/перезагрузке/спящем режиме

**Детекция приложений:**

Система поддерживает три режима обнаружения:

| Режим | Описание | Пример |
|-------|----------|--------|
| `PROCESS` | По имени процесса | `minecraft.exe` |
| `WINDOW_TITLE` | По подстроке в заголовке окна | Браузерные игры |
| `BOTH` | По имени процесса + проверка заголовка | Специфичные вкладки |

Для детекции по заголовку используется Win32 API `EnumWindows`, что позволяет обнаруживать браузерные игры и специфичный контент.

**Категории отслеживаемых приложений:** `GAME`, `LAUNCHER`, `BROWSER_GAME`, `SOCIAL_MEDIA`, `OTHER`.

### ClassroomGuardian

**Тип:** Windows-сервис (.NET Framework 4.7.2)

Служба-«сторож», обеспечивающая отказоустойчивость всего комплекса.

**Основные функции:**

- **Мониторинг процессов** — каждые 15 секунд проверяет, запущены ли ClassroomLogin и SessionWatcher; при необходимости перезапускает их
- **Контроль целостности** — сравнивает файлы в рабочей директории (`bin`) с резервной копией; при повреждении или удалении восстанавливает из бэкапа
- **Автообновление** — каждые 10 минут проверяет GitHub Releases на наличие новых версий; при обнаружении скачивает ZIP-архив, извлекает файлы и обновляет компоненты
- **Запуск в контексте пользователя** — сервис работает от имени SYSTEM, но запускает ClassroomLogin и SessionWatcher в сессии текущего пользователя через `CreateProcessAsUser` с дупликацией токена

**Механизм обновления:**

1. Запрос к GitHub API (`/repos/{owner/repo}/releases/latest`)
2. Сравнение `tag_name` из релиза с локальным `version.txt`
3. Скачивание ZIP-архива из assets релиза
4. Извлечение во временную папку, копирование exe/dll в `bin` и `backup`
5. Если в архиве есть `hosts.txt` — обновление системного файла hosts
6. Обновление самого Guardian через отложенный bat-скрипт

---

### ClassroomInstaller

**Тип:** Консольное приложение (.NET Framework 4.7.2)

Программа установки, запускаемая с правами администратора.

**Процесс установки:**

1. Проверка прав администратора
2. Остановка всех связанных процессов и служб
3. Очистка предыдущей установки
4. Создание структуры каталогов (`bin`, `logs`, `backup`)
5. Копирование исполняемых файлов в рабочую и резервную директории
6. Регистрация Windows-сервиса ClassroomGuardian (автозапуск, рестарт при сбое через 60 сек)
7. Настройка автозагрузки ClassroomLogin (реестр + планировщик задач)
8. Добавление записи в `hosts` для MySQL-сервера (`172.23.9.151  MySQL-8.4`)
9. Установка атрибутов Hidden+System на папку бэкапа
10. Создание скрипта деинсталляции (`uninstall.bat`)

## База данных

**СУБД:** MySQL 8.4
**Имя базы:** `classroom_db`

### Схема таблиц

| Таблица | Назначение |
|---------|------------|
| `sessions` | Журнал сессий пользователей (время входа/выхода, статус, heartbeat, IP) |
| `students` | Справочник студентов (группа, фамилия, имя) |
| `users` | Пользователи панели управления (логин, пароль, роль admin/teacher, привязка к кабинетам) |
| `tracked_apps` | Список отслеживаемых приложений (имя процесса, категория, способ детекции, паттерн заголовка) |
| `activity_log` | Лог обнаруженной активности (приложение, время начала/конца, длительность) |
| `remote_commands` | Очередь удалённых команд для рабочих станций |
| `sync_queue` | Очередь синхронизации оффлайн-операций |
| `active_sessions` | Представление (VIEW) — активные сессии с вычисляемым real_status |

### Статусы сессий

| Статус | Описание |
|--------|----------|
| `АКТИВЕН` | Сессия активна, heartbeat поступает |
| `ЗАВЕРШЁН` | Сессия корректно завершена |
| `АВАРИЙНЫЙ` | Сессия завершена некорректно (выключение питания, сбой) |

Представление `active_sessions` автоматически вычисляет `real_status = 'ПОТЕРЯНА СВЯЗЬ'`, если heartbeat не поступал более 120 секунд.

### Удалённые команды

| Тип команды | Описание |
|-------------|----------|
| `KILL_PROCESS` | Завершить процесс по имени (напр. `chrome.exe`) |
| `KILL_BY_TITLE` | Завершить процесс по подстроке в заголовке окна |
| `MESSAGE` | Отправить сообщение на рабочую станцию |
| `LOCK_SCREEN` | Заблокировать экран |

Команды имеют статусы: `PENDING` → `EXECUTED` / `FAILED` / `EXPIRED` (автоистечение через 5 минут).

## Установка

### Подготовка сервера

1. Установите MySQL 8.4
2. Создайте базу данных и пользователя:
   ```sql
   CREATE DATABASE classroom_db CHARACTER SET utf8mb4;
   CREATE USER 'classroom_user'@'%' IDENTIFIED BY '<пароль>';
   GRANT ALL ON classroom_db.* TO 'classroom_user'@'%';
   ```
3. Импортируйте схему из `classroom_db.sql`

### Установка на клиентские ПК

1. Скомпилируйте все проекты (ClassroomLogin, SessionWatcher, ClassroomGuardian, ClassroomInstaller)
2. Поместите все exe/dll/config файлы в одну папку
3. Запустите `ClassroomInstaller.exe` **от имени администратора**
4. Инсталлятор автоматически:
   - создаст структуру каталогов
   - зарегистрирует и запустит службу ClassroomGuardian
   - настроит автозагрузку ClassroomLogin
   - добавит запись MySQL-сервера в hosts

### Удаление

Запустите `C:\Program Files\ClassroomSystem\uninstall.bat` от имени администратора.

## Конфигурация

Основные параметры задаются в файлах `App.config` каждого компонента.

### Подключение к БД

| Параметр | Значение по умолчанию | Описание |
|----------|-----------------------|----------|
| `MySqlServer` | `MySQL-8.4` | Хост MySQL (через hosts) |
| `MySqlPort` | `3306` | Порт MySQL |
| `MySqlDatabase` | `classroom_db` | Имя базы данных |
| `MySqlUser` | `classroom_user` | Пользователь БД |
| `MySqlConnectionTimeout` | `3` (SessionWatcher) / `5` (Login) | Таймаут подключения, сек |

### Интервалы (SessionWatcher)

| Параметр | Значение | Описание |
|----------|----------|----------|
| `UpdateIntervalSeconds` | `30` | Частота heartbeat, сек |
| `SyncRetryIntervalSeconds` | `120` | Повтор синхронизации оффлайн-очереди, сек |
| `MaxRetryAttempts` | `5` | Макс. попыток перед удалением из очереди |
| `ActivityScanIntervalSeconds` | `10` | Частота сканирования процессов, сек |
| `TrackedAppsRefreshMinutes` | `60` | Обновление списка отслеживаемых приложений, мин |
| `CommandPollIntervalSeconds` | `10` | Опрос удалённых команд, сек |

### Интервалы (ClassroomGuardian)

| Параметр | Значение | Описание |
|----------|----------|----------|
| `CheckIntervalSeconds` | `15` | Проверка процессов и целостности, сек |
| `UpdateCheckMinutes` | `10` | Проверка обновлений на GitHub, мин |
| `GitHubRepo` | `owner/repo` | Репозиторий для автообновления |

## Автообновление

ClassroomGuardian поддерживает автоматическое обновление через GitHub Releases.

### Подготовка релиза

1. Скомпилируйте все компоненты
2. Обновите `version.txt` (формат: `1.2.3`)
3. Упакуйте все exe/dll/config файлы в ZIP-архив
4. Опционально: добавьте `hosts.txt` с записями для обновления системного файла hosts
5. Создайте GitHub Release с tag, совпадающим с версией, и прикрепите ZIP как asset

### Процесс обновления на клиенте

1. Guardian запрашивает `/repos/{repo}/releases/latest` через GitHub API
2. Сравнивает `tag_name` с содержимым локального `version.txt`
3. При несовпадении скачивает ZIP-asset, извлекает во временную папку
4. Останавливает ClassroomLogin и SessionWatcher
5. Копирует новые файлы в `bin` и резервную папку
6. Если в архиве есть `hosts.txt` — обновляет блок между маркерами `# --- ClassroomSystem BEGIN ---` / `# --- ClassroomSystem END ---`
7. Для обновления самого Guardian создаёт bat-скрипт, выполняемый при следующем запуске
